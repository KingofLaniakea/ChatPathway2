# 要从头到尾重新训练的实验计划

分成各个层级，部分可以组合。Excel 各列是一层，每行是一个可能并且推荐的实验组合：

- a 层：各个指引 latent dynamics 的类 HNN、PHNN、Neural ODE 网络类型；
- b 层：AE 训练相关实验，作为独立层级，探究怎样让 128 维 latent representation 更适合 HNN、PHNN 等网络；部分设置只适用于 a 层的部分网络；
- c 层：inference time 的区别。第一种是目前 Framework A 的直接 LoRA 推理；第二种是在此基础上生成多个答案再 rerank；第三种是 HNN rollout 与 LoRA 推理结果在 latent 层面加权融合，即以前的 Framework B；
- d 层：对比 dynamics 单独训练、先单独训练再与 SFT 联合训练，以及从一开始与 SFT 联合训练。

完整可筛选矩阵见 [实验矩阵.xlsx](实验矩阵.xlsx)。训练器参数、服务器命令和严格归因说明不放在本文，见 [agent 技术解读](agent/实验规划_技术解读.md)。

## 1. 四个实验层级

### A 层：latent dynamics 网络

| 编号 | 方法 | 作用 |
| --- | --- | --- |
| A0 | 不使用 dynamics | SFT 对照 |
| A1 | HNN：`dz/dt = J∇H` | 测试保守 Hamiltonian 结构 |
| A2 | forced-damped HNN：`dz/dt = (J-rI)∇H+F(t)` | 当前主方法，测试阻尼和显式时间强迫 |
| A3 | Neural ODE | 测试“有连续动力学”本身，而非 Hamiltonian 结构 |
| A4 | PHNN：`(J-R)∇H+G(z)u(t)` | 只有获得独立、可观测的控制量 `u` 后才能运行 |

### B 层：128 维 AE 表示

| 编号 | AE 设置 | 主要问题 |
| --- | --- | --- |
| B0 | 不使用 AE | 只适用于纯 SFT 对照 |
| B1 | 冻结 reconstruction AE | 当前基准：先重建，再冻结给 dynamics 使用 |
| B2 | reconstruction + 下一层预测一致性 | latent 是否更容易被连续动力学推进 |
| B3 | reconstruction + 几何条件约束 | latent 是否具有更稳定的尺度、协方差和 Poisson frame 条件数 |
| B4 | AE 与 dynamics 联合更新 | 联合适配是否优于固定 AE；必须与 B1 使用同一初始化对照 |

### C 层：推理方式

| 编号 | 推理方式 | 含义 |
| --- | --- | --- |
| C1 | direct LoRA | 只用第二次微调后的 LoRA 直接生成，即当前 Framework A |
| C2 | 多答案生成 + rerank | 生成多个完整候选，再用验证集冻结的评分器选择 |
| C3 | LoRA 与 dynamics latent 融合 | 在完整 JSON layer 边界融合 LoRA 状态与 rollout 状态，即 Framework B |

C3 不能按 token 盲目推进 graph-layer dynamics；必须先有能识别“一个完整 layer 已生成”的边界控制器。

### D 层：dynamics 与 SFT 的训练关系

| 编号 | 训练关系 | 用途 |
| --- | --- | --- |
| D0 | 第一阶段 SFT | 基础生成能力 |
| D1 | 第二阶段 SFT-only | 排除“只是又做了一次 SFT”的收益 |
| D2 | SFT → AE → dynamics-only，LoRA 冻结 | 测试 dynamics 能否单独学习 latent trajectory |
| D3 | 冻结 LoRA 预训练 dynamics → 低学习率联合训练 | 测试先让向量场在固定隐空间稳定，再联合适配是否更可靠 |
| D4 | SFT → AE → SFT+dynamics 直接联合训练 | 当前 Framework A 训练方式 |

D3 在新数据冻结后按同一套方法执行：固定第一阶段 SFT 和 AE，最多训练 dynamics 3 轮并按
validation 早停；随后联合训练，LoRA 学习率在 `5e-6` 与 `1e-5` 中预先选定，dynamics
学习率保持 `2e-4`。动力学对 LoRA 的影响从 0 缓慢升到目标权重，同时用原始 SFT 输出的
KL 约束限制漂移。训练保存“最佳 SFT”“最佳 dynamics”“最佳复合指标”三类 checkpoint，
但正式主结果只能使用预先指定的“最佳复合指标”checkpoint，另外两类只用于诊断。训练日志
记录 SFT 与动力学损失在 LoRA 参数上的梯度夹角。该配置对应 E011/E021，当前仍待实现，
不能标为已运行。

## 3. 运行顺序

当前代码可直接运行的第一批是 E000、E001、E012 和 E022：共享第一阶段 SFT/AE 后，比较
stage-1 SFT、同预算第二次 SFT、joint HNN、joint forced-damped HNN。它们分别对应机器矩阵中的
`exp000`、`exp003`、`exp001` 和 `exp002`。D2 dynamics-only、D3 先单训再联合、Neural ODE、
AE 表示消融以及 C2/C3 尚属于下一批实现，不能把表中的“推荐”误写成“已经可运行”。

1. 先固定同一份 SFT、split、seed 和 checkpoint，运行 E000–E031，确定 dynamics 与训练关系。
2. 只在胜出的 A/D 组合上运行 B2–B4，不同时更换推理方法。
3. 固定最佳 A/B/D 后比较 C1、C2、C3，并匹配候选数、生成 token 与计算时间。
4. PHNN 等到数据中出现独立、真实、带时序的 `u(t)` 后再运行。
5. 每个正式组合使用三个 seed，只报告 validation-selected checkpoint；不得跨 dataset revision 混比。

## 4. 第一轮核心结果

第一轮必须至少同时报告 E000、E001、E010–E012、E020–E022 和 E030–E031。这样才能分别判断：

- 改善是否只是来自第二次 SFT；
- HNN/forced-damped HNN 是否能在 LoRA 冻结时学到轨迹；
- 先训练 dynamics 再联合是否比直接联合更稳定；
- Hamiltonian 结构是否优于参数量和训练预算匹配的普通 Neural ODE。
