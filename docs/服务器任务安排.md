## 一次性准备

```bash
git fetch origin main
git merge --ff-only origin/main

python -m experiments.run_experiment prepare-data --overwrite
python -m experiments.run_experiment download-model

python -m experiments.validate_matrix

```

`prepare-data` 会从现有 171 GiB full train CSV 一次流式扫描，生成 pilot、
next-layer eval 和 multi-step eval，并执行 identity/schema/split 审计。不要
直接把 3225 万 prefix rows 当作第一轮训练集；否则长 pathway 会被按 prefix
数量过度加权，启动成本也不合理。

## GPU 冒烟

先用 `seed=999001, epochs=1, limit=32` 顺序跑 `base000`、`exp001`、
`exp002`，再用 `limit=2` 直接推理。确认日志中没有 NaN、没有 zero eligible
semantic layers，checkpoint 可以重新加载，输出含 JSON/schema validity。

## 正式矩阵

每个 seed 独立执行：

```text
base000 shared SFT -> shared AE
exp000 stage-1 direct baseline
exp003 compute-matched stage-2 SFT only
exp001 stage-2 SFT + HNN
exp002 stage-2 SFT + (J-rI) grad H + F(t)
```

正式 seeds：`20260711, 20260712, 20260713`。路径自动写到
`checkpoints/seeds/<seed>` 和 `runs/seeds/<seed>`；不同 seed 不覆盖，同 seed
三条 stage-2 臂严格共享 SFT/AE。

正式 wrapper 固定 `max_length=8192, batch_size=1`，SFT/AE/第二次 SFT/HNN/
forced-damped HNN 不得单独降配。该设置在 pilot 上保留 99.17% graph layer
和 91.07% substep，并已通过四卡 A100 冒烟。

验证集按 `pathway_family_id` 整族留出；SFT、AE 和三条 stage-2 臂对同一 seed
必须复用同一 family split，不能退回逐行或仅 `source_json` 划分。

首轮只做 direct greedy inference。`exp011`-`exp014` 已从可执行矩阵移除，
因为旧原型按 token 推进，而训练按 graph layer 推进。PHNN、Neural ODE 和
phenotype 因果实验也不插入首轮矩阵。

## 结果门槛

- stage-1 与 stage-2-only 是归因对照，不能只比较 HNN 与 FDHNN；
- 只汇报 validation-selected `checkpoint_best`；
- 汇总 3 个 seed，不用单 seed 最优值；
- Task 0/2 用 `method.analysis.semantic_latent_export` 导出同一表示；
- phenotype 缺失不算阴性；Task 4 没有真实干预和 validation-calibrated
  scorer 前不出准确率；
- Task 3 的 direction/shuffle/unrelated negatives 必须有人审/有 provenance。
