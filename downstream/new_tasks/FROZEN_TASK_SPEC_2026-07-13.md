# ChatPathway 下游任务定稿（2026-07-13）

本文件定格当前 Task 0–6 的科学问题、数据契约、评估指标与准备状态。
写作结构沿用原 `downstream task list.md`，但任务定义以本目录的实际评估
代码、`README.md` 和 `matrix.json` 为准。历史任务名称或编号若与本文件冲突，
以本文件为准。

## 总原则

1. 不同图层的先后来自有向通路图的拓扑方向，可解释为上游到下游的顺序，
   但不是有单位的物理时间，也不表示相邻图层时间间隔相等。
2. 同一图层内的原子事件默认是并行集合；没有额外因果证据时，不给同层事件
   强加内部顺序。
3. 表型缺失只统计缺失覆盖率并排除，绝不自动当作阴性。当前主训练数据不承担
   表型监督；Task 4 使用独立的真实干预数据。
4. 当前动力学中的 `F(t)` 只依赖归一化步进坐标，不是敲除、药物或其他显式
   干预输入。
5. HNN 的能量、向量场残差和轨迹误差默认只作诊断。未经验证集校准，不得把
   这些量当作因果正确性标签。
6. 验证集只用于阈值、权重和检查点选择；测试集冻结后不得用于调参。

## Task 0：AE/HNN 隐轨迹自洽性

### 核心问题

AE 是否保留了语言模型隐状态中的信息？学到的动力学能否从一个已观测图层状态
推进到后续真实图层状态？

### 评估单位与流程

1. 用同一个冻结的语言模型与 SFT 适配器提取提示锚点和每个图层的上下文化隐状态。
2. 用同一个冻结 AE 把 4096 维隐状态映射到 128 维隐空间并重建。
3. 从提示锚点或已观测图层开始，按图层数推进 HNN。
4. 在固定推进长度上比较重建状态、真实隐轨迹和动力学轨迹。

### 指标

- AE 重建均方误差与余弦距离；
- 各推进长度的隐状态均方误差、余弦距离；
- 有效样本覆盖率和因文本截断而缺失的图层数；
- 按 pathway family、图层长度和物种分层的误差曲线。

### 它能证明什么

它检验表示压缩和隐动力学是否在未见家族上具有可重复的轨迹拟合能力。

### 它不能证明什么

自洽性不等于生成文本正确，也不等于 128 维隐变量具有真实物理能量含义。

## Task 1：图层级原子事件集合条件续写

### 核心问题

给定前若干通路图层，模型能否预测下一个图层的原子 `A—关系—B` 事件集合，
并继续补全剩余图层？

### 数据单位

- 图层之间有上游到下游顺序；
- 同层的 `substeps` 按不计排列顺序的多重集合评分；
- 只有提供独立 `ordering_provenance` 时，才允许把原子事件当作扁平有序序列。

### 流程

1. 输入 `prefix_steps`。
2. 模型输出结构化 `remaining_steps`，每层包含 `step`、`layer` 和 `substeps`。
3. 统一规范化实体与关系。
4. 默认逐层对齐，再在层内做集合匹配。
5. 对旧 CSV 中由句子解析器恢复的边界单独报告覆盖率，不把解析失败静默算错。

### 指标

- 下一图层事件 Precision、Recall、F1、Exact Set Match；
- 剩余图层序列的层级匹配率；
- 实体匹配、关系匹配、完整三元组匹配；
- JSON 合法率、解析覆盖率、截断率。

### 它能证明什么

它检验局部生物关系与上游到下游的图层级续写能力。

### 它不能证明什么

它不能从拓扑顺序恢复真实反应时间，也不能证明同层事件存在先后。

## Task 2：预测—金标准隐轨迹误差（PCTE）

### 核心问题

模型生成答案的图层级隐轨迹，是否接近金标准答案在同一冻结表征下的隐轨迹？

### 流程

1. 对预测答案和金标准答案使用完全相同的基础模型、SFT 适配器、AE 与图层聚合规则。
2. 得到 `predicted_latents` 与 `target_latents`。
3. 因两条轨迹长度可能不同，使用动态时间规整对齐。
4. 计算对齐路径上的欧氏或余弦距离，并保存表征与检查点清单。

### 指标

- PCTE；
- 对齐路径长度和归一化误差；
- 按预测长度、真实长度和 pathway family 分层的误差；
- 无效 JSON 和无法抽取图层的覆盖率。

### 它能证明什么

它检验预测与金标准是否沿相近的图层级隐表示路径演化。

### 它不能证明什么

它不是 HNN 自洽性指标，也不能单独证明因果正确或生物事实正确。

## Task 3：专家验证的因果候选重排序

### 核心问题

面对一个正确通路候选及三类困难错误候选，模型能否把专家确认的正确候选排在
方向反转、图层扰乱和匹配无关通路之前？

### 正式候选类型

1. `positive`：恰好一个专家确认的正确候选。
2. `direction_reversal`：把有明确方向的关系反转后形成的错误候选。
3. `step_shuffle`：交换不同图层后形成的错误候选。
4. `unrelated_pathway`：在物种、领域和表面统计上匹配的无关通路困难负例。

### 交给数据构建与标注团队的具体指令

#### A. 样本选择和身份保留

1. 每例设置全局唯一 `id`，并保留：
   `source_json`、`organism`、`pathway_id`、`pathway_family_id`、
   `pathway_block`、`record_id`、`sample_id`。
2. 保存数据库条目、论文或其他证据来源，不得只保存改写后的自然语言。
3. 按 pathway family 和来源记录切分训练、验证、测试；同一家族、同一原始记录
   及其改写候选不得跨集合。
4. 测试集一经仲裁确认即冻结。

#### B. 正确候选

1. 每例只能有一个 `label: "positive"`。
2. 两名标注者独立判断关系方向、图层顺序和整体可成立性；分歧由第三名仲裁者处理。
3. 只有仲裁结束后才可设置 `expert_validated: true`。

#### C. 方向反转负例

1. 只反转有明确方向且反转后经专家确认不成立的关系。
2. 可逆反应、相关关系、双向调控和证据不足关系不得机械反转。
3. `negative_provenance` 必须记录 `construction_method`、
   `validation_protocol` 和所有 `reversed_relation_ids`。

#### D. 图层扰乱负例

1. 当前数据默认只允许 `shuffle_unit: "graph_layer"`，并交换不同图层。
2. 同层并行事件换序不是有效负例。
3. 必须保存 `original_order` 与 `shuffled_order`；两者包含同一组图层但顺序不同。
4. 扰乱后仍可能生物学成立的候选必须排除，不能只因文字顺序不同就标错。
5. 只有存在额外因果顺序证据时才允许 `shuffle_unit: "causal_substep"`，并记录
   证据来源与版本。

#### E. 匹配无关通路负例

1. 保存 `source_pathway_id` 和可复现的 `matching_protocol`。
2. 至少匹配物种、通路领域、图层数、文本或令牌长度、实体数、常见词汇、关系密度。
3. 避免模型仅凭物种名、长度或罕见词判断。

#### F. 标注验收

- 建议正式测试集每例同时包含三类负例；代码允许更少，但不能作为主结果设计。
- 报告双标一致率、仲裁比例、被排除的可逆/歧义关系数量和各负例类型覆盖率。
- 先冻结原始候选，再由评估程序计算 `llm_score`。人工团队不得填写或调整模型分数。
- HNN 诊断只允许 `hnn_rollout_error`、`hnn_vector_field_residual`、
  `energy_balance_residual`；原始能量升降不得当作正负标签。
- 若将 HNN 诊断并入排序，权重只能在验证集拟合并固定后用于测试集。

### 数据团队交付格式

```json
{
  "id": "case-000001",
  "question": "固定版本的候选判别问题",
  "expert_validated": true,
  "annotation_provenance": {
    "annotation_id": "annotation-batch-v1",
    "protocol_version": "task3-annotation-v1",
    "source_dataset_id": "source-dataset-v1"
  },
  "source_json": "...",
  "organism": "hsa",
  "pathway_id": "hsa00010",
  "pathway_family_id": "00010",
  "record_id": "...",
  "sample_id": "...",
  "candidates": [
    {
      "id": "gold",
      "text": "...",
      "label": "positive"
    },
    {
      "id": "reverse-1",
      "text": "...",
      "label": "negative",
      "negative_type": "direction_reversal",
      "negative_provenance": {
        "construction_method": "structured edge reversal",
        "validation_protocol": "task3-annotation-v1",
        "reversed_relation_ids": ["relation-7"]
      }
    }
  ]
}
```

评估时由冻结模型追加 `llm_score`，必要时再追加验证集校准后的 HNN 诊断组合分数。

### 指标

- Top-1、MRR；
- 总体及分负例类型的成对拒绝率；
- 可选的验证集校准组合分数；
- 覆盖率、专家一致率和各困难负例的匹配质量。

### 它能证明什么

它检验模型是否识别关系方向和跨图层顺序，而不是只复述实体集合。

### 它不能证明什么

它不能把 HNN 能量直接解释为因果方向，也不能替代真实干预实验。

## Task 4：真实敲除与救援推断

### 核心问题

在有真实实验依据的 WT、KO/KD/抑制和救援条件下，模型预测的表型概率能否匹配
观测终点，并把真实救援干预排在无效救援之前？

### 交给数据构建与标注团队的具体指令

#### A. 数据来源

1. 只纳入有真实干预与测量终点的实验；纯知识图谱推断或人工假设不能作为金标准。
2. 每例保存：物种、组织、细胞系、数据库 accession、论文 DOI/链接、干预方式、
   靶点、剂量、时间点、表型测量方法、生物重复数、效应方向与统计证据。
3. WT、KO 和救援应来自同一或明确可比的实验背景；不满足者标为不可比较并排除。

#### B. 表型定义

1. 为每个 `phenotype_id` 写出可执行的 `positive_definition`，包括终点、阈值、
   时间窗口和方向。
2. 无表型标注的记录只能交付：

```json
{"case_id": "case-unknown-1", "phenotype_available": false}
```

3. 缺失不得当作阴性，也不得添加虚构 WT/KO 状态。

#### C. 状态与干预

1. 每例必须恰好一个 `role: "wt"`，且 `interventions` 为空。
2. 每例至少一个 `role: "ko"`，干预类型只能是 `knockout`、`knockdown` 或
   `inhibition`。
3. 可选 `role: "rescue"` 必须通过 `parent_ko` 指向一个 KO，保留父 KO 干预，
   并额外加入 `overexpression`、`activation` 或 `drug`。
4. 为评估救援排序，每个父 KO 最好配置多个候选，至少一个真实救援和一个无效对照。
5. `gold_positive` 只能来自冻结的实验终点定义。

#### D. 切分和防泄漏

1. 同时按 pathway family、论文/实验批次和近重复干预组合隔离。
2. 同一实验的 WT、KO、救援必须留在同一个集合中。
3. 表型评分器的阈值和概率校准只在验证集完成；测试集不得选择阈值。

#### E. 数据团队与模型团队边界

1. 数据团队交付实验事实、状态、`gold_positive` 与证据，不填写
   `predicted_probability`。
2. 模型团队冻结基础模型、适配器、动力学检查点和提示模板后生成概率。
3. 当前模型统一使用 `dynamics_conditioning: "prompt_initial_condition"`：干预写入
   提示并改变初始隐状态。
4. 只有未来专门训练显式干预条件动力学后，才可使用
   `explicit_intervention_conditioned_dynamics`。不得把当前 `F(t)` 称为敲除输入。

### 数据团队交付的中间格式示意

```json
{
  "case_id": "case-000001",
  "phenotype_available": true,
  "dataset_provenance": {
    "dataset_id": "intervention-dataset",
    "dataset_version": "v1",
    "split": "test",
    "evidence_source": "accession/DOI"
  },
  "phenotype": {
    "phenotype_id": "survival-24h",
    "positive_definition": "24 h survival above the frozen assay threshold"
  },
  "context": {
    "organism": "human",
    "cell_line": "...",
    "dose": "...",
    "timepoint": "24 h"
  },
  "states": [
    {
      "state_id": "wt",
      "role": "wt",
      "interventions": [],
      "gold_positive": 1
    },
    {
      "state_id": "ko-gene-a",
      "role": "ko",
      "interventions": [{"kind": "knockout", "target": "GENE_A"}],
      "gold_positive": 0
    }
  ]
}
```

模型评估转换程序再追加 `model_provenance`、`phenotype_scorer`、
`dynamics_conditioning` 和每个状态的 `predicted_probability`，形成评估器要求的最终
JSONL。

### 指标

- 表型概率 Brier 分数与阈值后准确率；
- KO 效应方向准确率；
- 救援 Hit@1、MRR 和恢复比例；
- 有表型、无表型、不可比较实验的覆盖率；
- 按物种、细胞背景、干预类型和 pathway family 分层结果。

### 它能证明什么

它是在真实干预金标准下检验反事实表型与救援排序的任务。

### 它不能证明什么

它不能用主训练数据中缺失的表型补标签，也不能把时间外力 `F(t)` 解释成具体药物
或敲除控制量。

## Task 5：单细胞扰动迁移

### 核心问题

经过明确单细胞适配的模型能否预测未见细胞或未见扰动的表达响应？通路/HNN 先验
是否在受控条件下带来额外收益？

### AutoDL 到 CFFF 的迁移清单

迁移时必须成套保存并校验：

- 原始或冻结版本 AnnData；
- 基因顺序、细胞标识、扰动标签和对照匹配表；
- 训练、验证、测试切分；
- 归一化、过滤、细胞句子构建脚本与参数；
- 数据来源、版本、许可证和 SHA-256；
- 已有单细胞适配器、预处理检查点及其训练配置和日志；
- 不使用通路/HNN 先验但容量和优化设置相同的对照模型。

最终评估 NPZ 使用相同形状的 `[cell, gene]` 数组：`control`、`observed`、
`predicted`，可选 `baseline_predicted`。清单必须冻结基因/细胞顺序、扰动 ID、
归一化、对照匹配、切分和全部检查点身份。

### 指标

- 全基因与差分表达的 Pearson、Spearman；
- Top-DE 基因相关性；
- `predicted-control` 对 `observed-control` 的相关性；
- 相对受控基线的增量；
- 见过/未见扰动、见过/未见细胞类型的分层结果。

### 它能证明什么

它检验通路表示能否迁移到真实单细胞扰动响应。

### 它不能证明什么

未经单细胞适配，不能声称通路检查点可直接读取 AnnData；没有匹配对照也不能把
提升归因于 HNN 或通路先验。

## Task 6：BioMaze 独立机制问答

### 核心问题

冻结检查点能否在独立的生物机制选择题基准上选择正确答案？

### 下载和冻结要求

下载后先保存：

- 官方来源、版本、许可证、原始文件校验和；
- 官方验证/测试切分和任何许可的示例训练集；
- 原始问题、选项、金标准、标签和预处理脚本；
- 模型检查点训练时间线；
- 训练语料污染检查方法、状态和结果。

不得在测试题上微调、选择检查点或调提示模板。只有检查点时间线和污染审计支持
时才能报告“零样本”；否则按“外部冻结基准评估”表述。

### 最终记录字段与指标

每题保存 `id`、`question`、`choices`、`gold_option`、`predicted_option`，可选
`tags`。报告总体准确率、答案合法率、无效输出率和按标签分层准确率。

### 它能证明什么

它检验模型在外部机制问答上的选择能力和答案格式稳定性。

### 它不能证明什么

若污染审计未完成，不能把高准确率解释为未见知识上的机制推理能力。
